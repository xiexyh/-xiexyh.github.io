# 软件包准备

下载好 zookeeper-3.4.10 的

# 进入软件包的目录下解压安装压缩包将 zookeeper 安装包解压缩到 /opt 目录下

在终端执行命令： cd /opt/hadoop-package
在终端执行命令： tar -zxvf zookeeper-3.4.10.tar.gz -C /opt/

解压完成截图：

# 准备文件

  • 进入 zookeeper 配置文件目录
    在终端执行命令： cd /opt/zookeeper-3.4.10/conf
  • 将 zoo_sample.cfg 配置文件复制一份并重命名为 zoo.cfg
    在终端执行命令： cp zoo_sample.cfg zoo.cfg
  • 创建存放数据的文件夹
    在终端执行命令： mkdir /opt/zookeeper-3.4.10/data
  • 创建 myid 文件，并加入数字 1 （hadoop1 主节点为 1、hadoop2 从节点为 2、hadoop3 从节点为 3）
    在终端执行命令： vim /opt/zookeeper-3.4.10/data/myid

# 修改 zoo.cfg 配置文件

在终端执行命令： vim /opt/zookeeper-3.4.10/conf/zoo.cfg
将配置文件中 dataDir 的路径改为 /opt/zookeeper-3.4.10/data

修改好的配置截图：

zoo.cfg 各语句的含义
tickTime ：服务器与客户端之间交互的基本时间单元（ms）
initLimit ：允许 follower 连接并同步到 leader 的初始化时间，它以 tickTime 的倍数来表示。当超过设置倍数的 tickTime 时间，则连接失败
syncLimit ：Leader 服务器与 follower 服务器之间信息同步允许的最大时间间隔，如果超过次间隔，默认 follower 服务器与 leader 服务器之间断开链接
dataDir ：保存 zookeeper 数据路径
clientPort ：客户端访问 zookeeper 时经过服务器端时的端口号

# 同步配置文件到 hadoop2 和 hadoop3 上

  • 将配置文件同步到 hadoop2 上
    在终端执行命令： scp -r /opt/zookeeper-3.4.10/ root@hadoop2:/opt
  • 将配置文件同步到 hadoop3 上
    在终端执行命令： scp -r /opt/zookeeper-3.4.10/ root@hadoop3:/opt

# 修改 hadoop2 和 hadoop3 上的 myid

在 hadoop2 上将 myid 修改为 2
在终端执行命令： vim /opt/zookeeper-3.4.10/data/myid

在 hadoop3 上将 myid 修改为 3
在终端执行命令： vim /opt/zookeeper-3.4.10/data/myid

# 启动 zookeeper 集群并测试

# 分别启动 hadoop1、hadoop2、hadoop3 节点的 zookeeper 集群命

在 hadoop1 启动 zookeeper
在终端执行命令： zkServer.sh start

在 hadoop2 启动 zookeeper
在终端执行命令： zkServer.sh start

在 hadoop3 启动 zookeeper
在终端执行命令： zkServer.sh start

# 查看启动成功命令

通过 zkServer.sh status 命令可以查看启动的状态

至此，zookeeper-3.4.10 已经成功安装部署啦
