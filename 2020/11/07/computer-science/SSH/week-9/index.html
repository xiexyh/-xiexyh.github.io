# 子查询（HQL）

  • all
    语法：all (select 子查询)
    含义：子查询结果全部
  • any
    语法：any (子查询)
    含义：子查询结果中至少有一条

# HQL 查询优化

（1）避免不恰当的 or 操作
（2）避免使用 not
（3）避免 like 的特殊形式
（4）避免 having 子句
（5）避免 distinct
（6）避免让 索引 失效的情形

# 用注解实现持久化类、关联关系

在 Java 类中 添加若干注解来表示以下信息：
（1）Java 中声明持久化类 @Entity
（1）Java 类与表的对应关系 @Table
（2）Java 类主属性与表主键的对应关系 @Id
（3）Java 类一般属性与表一般属性的对应关系 @Column

好处：
不再需要 Java 类.hbm.xml 配置文件，简化了开发过程。
配置映射关系的常用注解

注解含义和作用
@Entity该类是一个持久化类
@Table指定对应的表 （默认值：持久化类名，不带包名）
@Id指定主键
@Column指定非主键列。
注：无此注解时，默认对应到表中的同名字段
@GeneratedValue指定主键生成策略。 注：无此注解时，默认为 assigned。
@GeneratedValue (strategy=GenerationType.AUTO) // 相当于 native
@UniqueConstraint唯一值约束
@Notnull非空约束
@Lob该属性对应 Blob 类型、Clob 类型
@Transistent该属性无需持久化到数据库中（即在数据库中没有字段与之对应）

如何使用注解？：
（1）在 Java 类中，Import javax.persistence.*;
（2）在 Java 类中，使用注解表示 Java 类与表、Java 属性与表字段等对应关系。
配置关联映射的常用注解

注解作用
@OneToOne对象 A 和对象 B 是 一对一关系
@OneToMany对象 A 和对象 B 是 一对多关系
@ManyToOne对象 A 和对象 B 是 多对一关系
@ManyToMany对象 A 和对象 B 是 多对多关系

结论：
有了注解，没有不必再写映射文件了。直接在 java 类中写出表映射、字段映射关系即可。
