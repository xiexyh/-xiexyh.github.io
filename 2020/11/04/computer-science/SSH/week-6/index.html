# 课堂笔记

# 主键生成策略

  • native
    由 Hibernate 根据数据库的类型自动生成主键。Java 程序不必管理主键。
    使用频率：常用
    优点：不管底层使用何种数据库，都适用。
  • assigned
    Java 程序要生成主键值，Hibernate 不生成主键值。（默认方式）
    使用频率：常用
  • increment
    主键由 hibernate 维护。
    缺点：；两个用户同时插入记录时，可能发生主键冲突
    使用频率：较少
  • identify
    主键由数据库维护。适用于 mysql 、 sql server 等数据库
    它要求数据库把主键定义为自动增长字段类型
    优点：无线程冲突问题。
    使用频率：较少
  • sequence
    主键由数据库维护，适用于 Oralce 。
    由 sequence 自动生成值来作为主键值。
    使用频率：较少

# 持久化对象的三种状态

持久化对象：PO = VO + hbm 映射
PO 对象的三种状态
划分依据：是否与 session 关联（.save (obj) .update (obj) .get (obj)）、是否有 OID

是否与 session 关联是否有 OIDJava 对象的状态对表中记录的影响力
××瞬时态无法影响表中的记录
×√游离态只有在关联上 session 后，才能影响表中记录
√√持久态能够影响到表中记录
当发生 session.getTransaction ().commit (); 表中的记录就会与被 Java 对象更新。
  • 瞬时状态（transistent）
    本质特征：无 OID、且未与 session 关联
    例如以下两种情况：
    情况 (1) new 出来的对象。
    情况 (2) 被 session.delete(obj) 的对象。
  • 持久化状态（Persistent）
    本质特征：有 OID、且与 session 关联
    例如以下两种情况：
    情况（1）从表中查到对象： session.get() 、 session.load()
    情况（2）使用对象更新表中的记录： session.save(obj) 、 .update(obj) 、 .saveOrUpdate(obj) 、 .merge(obj)
  • 游离状态（Detached）
    本质特征：有 OID、且未与 session 关联
    例如以下情况：
    session.close() ，引起 obj 与数据库的连接断开。

# 脏检查、刷新缓存

session 缓存
作用： load 、 get 执行后，查出的对象的所有属性会被记录在缓存，形成快照。
当后来发生 查询 、 事务提交 、 session.flush() 时，hibernate 会将涉及的对象与快照进行比较，相同，则不更新数据库。不同，才更新数据库。
缓存的目的，是减少访问数据库的次数。
缓存的存在时长:
只要 session 不关闭，对象就一直存在于缓存中。
缓存的常见操作

序列方法作用
1session.flush()用缓存中的对象更新数据库
3session.clear()清空缓存中的所有对象
3session.evict(object)清空缓存中的指定对象
4session.refresh(object)让对象回到快照的状态（刚从数据库中 get 到的状态）
  • 脏检查
    （1）使用 session.load 、 session.get 从数据库中查到对象时，session 会在缓存中记住该对象的所有属性（称为快照）。
    （2）执行 transaction.commit() 时，hibernate 会检查对象与快照是否完全相同。
    把事务提交时的对象与快照进行比较，称为 脏检查 。
    检查的目的：如果对象与快照一致，则无需更新数据库，节省时间、资源。只有当对象与快照不一致时，才需要更新数据库。
  • 刷新缓存
    含义：用 session 缓存中的对象属性，去更新数据库
    何时发生：
    (1) session.flush()
    (2) transaction.commit()

# Hibernate API 更新数据

更新数据的 三种 方法：

  • session.update（游离态对象）
    功能：用游离态对象 去更新表中对应记录。
  • session.saveOrUpdate（瞬时态对象 | 游离态对象）
    功能：
    当传入瞬时态对象时，效果相当于 session.save ()，即将对象插入到表中
    当传入游离态对象时，效果相当于 session.update ()，用对象去更新表中对应记录。
  • session.merge（瞬时态对象 | 游离态对象）
    功能:
    当传入的是瞬时对象，则将该瞬时对象插入数据库，同时返回持久态的该对象。
    当传入的是游离态对象，则根据该对象更新数据库，同时返回持久态的该对象。

merge 与 save 方法的比较

  • 相同点： 都是将 new 出来的对象 保存到表中
  • 不同点：merge 保存后会返回该 PO 对象，而 save 不能返回该 PO 对象。
    merge 与 update 方法的比较
  • 相同点：都是用游离态的对象去 更新表中的记录
  • 不同点：merge 更新后会返回该 PO 对象，而 update 返回 void 值。
    总结：
    存入用 save
    更新用 update
    如果存入、更新后还要使用该 PO 对象，则使用 merge
