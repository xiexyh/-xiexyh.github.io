# Java 类中表示三种关联

1、一对一
双向关联：
从 A 对象可以查到 B 对象，同时从 B 对象也能查 A 对象，则称为 双向关联 。
例：人与身份证

单向关联：
从 A 对象能查到 B 对象，但从 B 对象查不到 A 对象，则称为 单向关联 。
例：人与身份证

2、一对多
例：部门与员工

3、多对多
例：学生和课程

# 加载策略

使用 Hibernate 在两个 Java 类建立了关联映射后，可以方便地从一个 Java 对象查到另一个 Java 对象的信息。
例如，在 Dept 对象和 Emp 对象之间建立了一对多关联映射后，可以很方便地从一个 dept 对象查到所有 emp 的信息。
假如业务实际上只要用到 Dept 对象，而程序却在加载 Dept 对象时，把 Dept 对象关联的所有 emp 对象也加载上了，就会产生两个弊端：
（1）执行不必要的查询语句，降低程序性能
（2）查出来的对象占据额外的内存空间
为了克服以上弊端，Hibernate 采用了 延迟加载的策略，只有当真正需要用到关联的对象，才会加载关联的对象。

级别取值
class 标签中设置 lazy= "true"true 延迟加载
false 立即加载
set 标签中设置 lazy ="true"true 延迟加载
extra 增强延迟加载
false 立即加载
many-to-one 标签中设置 lazy ="proxy"proxy 延迟加载
no-proxy 无代理延迟加载
false 立即加载

为了使程序响应时间尽量短、占用内存空间尽量少， hibernate 在上述三个级别上采用的都是 延迟加载策略。
