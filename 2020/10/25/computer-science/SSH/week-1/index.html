# Oracle 中的数据类型

Oracle 中的数据类型： 字符型 、 数值型 、 日期型 、 LOB类型 、 （BLOB、CLOB）

  • 字符类型
    （1）以 固定长度 存放字符串：CHAR (n)、NCHAR (n)
    说明：CHAR (10)，存一个字符串 ’abc’

    区别：NCHAR 以 UNICODE 存储每个字符
    n 的最大取值：

    （2）以 可变长度 存储字符串 VARCHAR2 (n)、NVARCHAR2 (n)
    说明：VARCHAR2 (10)，存一个字符串’abc’

    区别：NVARCHAR2 以 UNICODE编码 存储每个字符
    n 的最大取值：
  • 数值型
    从输入的数值划分： 整数 、 小数
    （1）NUMBER 类型
    a. 表示小数

    举例： NUMBER (5,2)，总共 5 个有效数字，小数点后保留 2 位，例如：123.45
    b. 表示整数 NUMBER (P)
    举例：NUMBER (5)，5 位数的整数
  • 日期型
    • （1）DATE 型
      格式： YYYY-MM-DD HH:MM:SS
      精确程度： 秒
    • （2）TIMESTAMP
      精确程度： 0.000001秒
      但使用较少
  • 大对象类型（Large OBject）
    • （1） BLOB 类型 (Binary Large OBject)
      以 二进制 形式存储
      举例：图片、音频、视频
      最大存储 4G 字节
    • （2） CLOB 类型 (Character Large Object)
      以 字符串 形式存储
      举例：简历、新闻、文章
      最大存储 4G 字节

# 数据类型总结

图片、音频、视频BLOB

# 约束

6种 常用约束

约束关键字
主键约束primary key
外键约束foreign key
非空约束not null
唯一值约束unique
默认约束default ‘默认值’
Check 约束举例：check (gender=' 男 ' or gender=' 女 ');

2个 添加约束的时机

  • （1）创建时添加约束
  • （2）创建表之后，使用 alter table 命令
    • 添加主键约束
    • 添加外键约束

使用 PL/SQL Developer 图形界面添加约束

# ROWID 和 ROWNUM

  • ROWID
    说明：RowID 可以理解为 Oracle 为每一张表添加的一个 隐藏列 。
    表中的每一行（记录），都有一个行地址，ROWID 用于查询该行地址。
    任意两行的 ROWID 不同。
    ROWID 只能 查询 ，不能 修改 、 删除 。
  • ROWNUM
    说明：对于查询结果表，rownum 能返回每一行在结果表的序号。
    结果表中的第一行 rownum 为 1、第 n 行 rownum 为 n。
